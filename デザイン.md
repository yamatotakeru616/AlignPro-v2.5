# AlignPro CIM デザイン・パフォーマンス定義書 (デザイン.md)

本ドキュメントは、3D道路線形プロトタイプ（AlignPro v2.5）におけるUIデザイン、操作性、およびNVIDIA 3050 Ti (VRAM 4GB) 制限を考慮した動作負荷（パフォーマンス）の全タブ共通化および一括制御について定義します。

---

## 1. UIビジュアル＆操作性の共通定義
本アプリは日本のi-ConstructionおよびBIM/CIMに準拠しながらも、SF調のモダンな「Cosmic Slate」を基調テーマとしています。

- **配色（Color Palette）**:
  - ベース背景: `#050508` (ディープ・ cosmic ブラック)
  - パネル/カード: `bg-slate-950/95` (ガラスモフィズム半透明) + `backdrop-blur-md`
  - ボーダー: `border-white/10` または `border-slate-800`
  - アクセント1（アラインメント基準）: `text-blue-400` / `bg-blue-600`
  - アクセント2（確認・プレビュー）: `text-emerald-400` / `bg-emerald-600`
  - 警告/注意/土工（盛土）: `text-red-400` / `bg-red-600`

- **操作性の統一**:
  - トグルスイッチやポップアップ、ミニマップには共通のイージングアニメーション（`transition-all duration-300 transform`）を適用。
  - すべてのカード・パネルの角丸は `rounded-xl` または `rounded-2xl` で統一。
  - すべてのモーダルは `fixed inset-0 z-50 flex items-center justify-center p-4 backdrop-blur-sm` で一貫。

---

## 2. 統合パフォーマンス一括制御モード（全タブ連動）
ローカルPCやリソース制限のある環境（VRAM 4GB等）でブラウザがハングアップしたり動作が重くなったりするのを防ぐため、アプリ全体の描画負荷を一括制御する「パフォーマンス・モード」を定義・実装します。

### ■ 3つの統合パフォーマンス・モード

| モード名 | 指定値 | 概要・3D制御 | 2D/地図制御 |
| :--- | :--- | :--- | :--- |
| **超軽量（VRAMエコ）** | `'eco'` | 影（Shadow）OFF、アンチエイリアス（AA）OFF、テクスチャ簡素化、3D車両・地形メッシュ頂点数を極限まで削減。 | 2Dコンター（等高線）の自動間引き・非表示、アニメーションを一部停止。 |
| **標準（バランス）** | `'standard'` | 影ON（解像度低）、アンチエイリアスON、標準3Dマテリアル（MeshStandardMaterial）、中解像度地形。 | 標準コンター（等高線）5m間隔表示、UIアニメーションON。 |
| **高品質（フルスペック）** | `'high'` | リアルタイム高品質影（Soft Shadow）、アンチエイリアスON（超精細）、ライティング効果最大、高精度自動車モデル。 | 高密度コンター（1m間隔）、高速物理計算、パーティクルエフェクトON。 |

## 3. 実装の設計方針（一括切り替え）
- `App.tsx` において、アプリ全体の共通ステート `performanceMode` を用意。
- ヘッダー部分に、トグルまたはセレクター（エコ、バランス、高品質）を配置し、クリック一発で一括切り替え可能とする。
- このステートを `MapTab`、`Preview3DTab`、`DrawingsTab` などの各子コンポーネントに Props として伝播させ、以下のように負荷を動的にコントロールする：
  - **MapTab**: `contourInterval` 等高線描画解像度の最適化（eco時は等高線レンダリングをパスして高速化）。
  - **Preview3DTab**: `Three.js` の `WebGLRenderer` における影（ShadowMap）の有無、地形ジオメトリ（PlaneGeometry）の頂点分割数、自動車モデル（ヘッドライトの動的影等）の品質スイッチ、アニメーションのフレームレート上限。
  - **DrawingsTab**: リアルタイムSVGの簡易プレビューハッチング密度の調整。
- ユーザーの選択したパフォーマンス・モードはブラウザの `localStorage` に自動的に保存・永続化され、次回起動時にも同一のモードで起動。

## 4. 機能一覧との同期
